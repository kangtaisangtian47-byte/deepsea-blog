LubuntuでHugoブログを運営する際に、表示の更新や公開作業で迷わないためのコマンド集。

## 1. 執筆・プレビュー（確認用）

記事を書きながらブラウザでリアルタイム確認するためのコマンド。

> **hugo server -D**

下書き（draft: true）の記事も含めて表示されます。ブラウザで **http://localhost:1313** を開いて確認する。

## 2. 公開用データの生成（ビルド）

古いキャッシュをクリアしてから、本番用のファイルを生成する確実な手順。

> **rm -rf public/ && hugo**

これで public/ フォルダ内に最新の静的ファイルが書き出される。

## 3. Gitで公開・バックアップ（重要）

作成した記事や設定をGitHubなどに反映させる一連の流れ。

> **git add .**

（すべての変更をステージングに上げる）

> **git commit -m "記事追加：タイトルなど"**

（変更内容にメモを付けて保存する）

> **git push origin main**

（ネット上のリポジトリに送信し、公開を反映させる）

## 4. トラブル時のキャッシュクリア

もし挙動がおかしい時は、Hugo内部のキャッシュを掃除する。

> **hugo mod clean**

## まとめ（運用ルーチン）

1. **確認**：hugo server -D
2. **ビルド**：rm -rf public/ && hugo
3. **公開**：git add . -> commit -> push

これだけ手元に控えておけば、記事の更新作業で止まることはない。
