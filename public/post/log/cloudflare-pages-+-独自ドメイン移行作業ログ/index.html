1. ドメイン取得とDNS初期設定

  • レジストラで独自ドメインを取得。
  • Cloudflareへドメインを追加し、ネームサーバーを切り替え。
  • Search Console認証用のTXTレコードをDNSに登録し、所有権を確認。

2. 静的サイトジェネレーター側の構成変更

  • 設定ファイル（hugo.toml 等）の baseURL を新ドメインに書き換え。
  • リポジトリを更新し、ホスティング先へデプロイ。

3. ホスティング側カスタムドメイン紐付け

  • 管理画面からカスタムドメインを追加。
  • ステータスが Verifying（検証中）から進まない問題が発生。
  • DNSレコード内の古い「タイプA」レコード（初期設定のIPアドレス）を削除。
  • 替わりにルートドメインをホスティング側のドメインへ向けるCNAMEレコードを手動追加。
  • ステータスが Active に移行。

4. インデックス最適化

  • Search Consoleにて sitemap.xml を送信。
  • ドメイン浸透待ちによる一時的なエラーを確認。24時間以内の反映を待機。

感想

ドメインの紐付けにおいて、デフォルトで生成されるAレコードと追加したCNAMEが競合し、検証が止まる現象に時間を取られた。

UI上の「Verifying」という表示だけでは解決策が直感的ではないが、不要なレコードを削るという基本操作で解消。

インフラ周りの設定は一度完了すれば自動化されるため、今後はコンテンツ制作にリソースを割く。
